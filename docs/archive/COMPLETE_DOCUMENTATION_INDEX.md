## 🎯 Common Workflows

### Workflow 1: UI/UX Review with Antigravity

```bash
# Step 1: Read guide
Open docs/UIUX_ANTIGRAVITY_REVIEW.md

# Step 2: Capture screenshots
15 screenshots total (desktop + mobile for each page)

# Step 3: Send to Antigravity
- Upload UIUX_ANTIGRAVITY_REVIEW.md
- Upload all screenshots
- Wait for analysis

# Step 4: Review results
- Go through each page analysis
- Note critical issues
- Create action plan

# Step 5: Implement fixes
- Fix critical first
- Then high priority
- Then medium/low

# Step 6: Re-test
- Capture new screenshots
- Send to Antigravity again
- Compare scores
```

---

### Workflow 2: Adding New Feature

```bash
# Step 1: Check roadmap
Open docs/COMPREHENSIVE_IMPROVEMENT_PLAN.md

# Step 2: Check current phase
See what phase you're in

# Step 3: Implement feature
Follow coding standards from AI_CONTEXT.md

# Step 4: Test
Run: bun run lint
Run: bun run build
Manual test

# Step 5: Document
Update CHANGELOG.md
Add to relevant docs

# Step 6: Commit
git add .
git commit -m "feat: description"
git push
```

### Workflow 3: Fixing Bugs

```bash
# Step 1: Identify issue

bun run lint
```

## 🎓 For AI Agents

### When Working on This Project:

**1. Read First:**
- `AI_CONTEXT.md` - Essential guide
- `CHANGELOG.md` - Recent changes
- `COMPREHENSIVE_IMPROVEMENT_PLAN.md` - Roadmap

**2. Follow Standards:**
- TypeScript strict mode
- Tailwind v4 syntax
- Components <300 lines
- Add comments for complex logic

**3. Test Before Committing:**
```bash
bun run lint
bun run build
Manual test
```

**4. Document Changes:**
- Update `CHANGELOG.md`
- Add to relevant docs
- Commit with conventional commits
